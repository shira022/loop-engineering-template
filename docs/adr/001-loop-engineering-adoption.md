# ADR-001: ループエンジニアリングの採用

## ステータス
accepted

## コンテキスト
AIエージェントを活用したソフトウェア開発において、
エージェントが過去の経験から学習し、継続的に改善する仕組みが必要。

## 決定
ループエンジニアリングを採用する。
- 全てのセッションで学習サイクルを回す
- 知識は learnings/ に蓄積
- 繰り返しパターンはスキル化
- アーキテクチャ判断はADR化

## 理由
- エージェントの作業品質が継続的に向上する
- プロジェクト知識が暗黙知にならず文書化される
- 新しいエージェントも過去の知見を活用できる

## 結果
- learnings/ ディレクトリで知識を管理
- .agents/skills/ でスキルを管理
- docs/adr/ で決定を管理
